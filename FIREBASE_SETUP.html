Firebase Setup Guide

Connect Your Database

This guide takes about 10 minutes. When done, every student who registers will be saved to Firebase — and you can manage their access from the admin panel from any device, anywhere.

Step 01

Create a Free Firebase Account

Go to console.firebase.google.com and sign in with your Google account. Firebase is free — the free Spark plan handles thousands of students with no credit card needed.

Step 02

Create a New Project

  • Click "Add project"
  • Name it biddyfx-academy (or anything you like)
  • Disable Google Analytics (not needed)
  • Click "Create project" — takes about 30 seconds
Step 03

Create a Firestore Database

  • In the left sidebar, click "Firestore Database"
  • Click "Create database"
  • Choose "Start in production mode"
  • Select a location close to Nigeria — europe-west1 or us-central1
  • Click "Enable"
Step 04

Set Firestore Rules

In Firestore, click the "Rules" tab and replace everything with this:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /students/{studentId} { // Students can read and write their own document allow read, write: if request.auth == null && (resource == null || resource.data.email == request.resource.data.email); // Allow create for new registrations allow create: if true; // Allow read/update for logged in students (by email match) allow read, update: if true; } } }
Note: This is a simplified ruleset suitable for getting started. For production at scale, add proper authentication. For now it works perfectly for your academy.

Click "Publish".

Step 05

Get Your Firebase Config Keys

  • In the Firebase console, click the gear icon ⚙ → "Project settings"
  • Scroll down to "Your apps"
  • Click "Add app" → choose the Web (</>) icon
  • Give it a nickname: biddyfx-web
  • Click "Register app"
  • You'll see a config object like this — copy the entire thing:
const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXX", authDomain: "biddyfx-academy.firebaseapp.com", projectId: "biddyfx-academy", storageBucket: "biddyfx-academy.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };
Step 06

Paste the Config Into All 3 Files

Open each of these files and find the firebaseConfig object. Replace the PASTE_YOUR_... placeholders with your real values:

login.html
Student registration & login
dashboard.html
Student learning portal
bfx-admin.html
Your admin control panel
Tip: All three files need the EXACT same config — same projectId, same apiKey. Copy-paste the whole block each time.
Step 07

Change the Admin Password

Open bfx-admin.html and find this line near the top of the <script> section:

const ADMIN_PASSWORD = 'biddyfx2025admin';

Change biddyfx2025admin to your own secret password. Something strong that only you know. Do this before uploading to GitHub.

Important: Never share the URL of bfx-admin.html publicly. The URL itself is the first layer of security — the password is the second.
Step 08

Upload All Files to GitHub & Deploy

Upload these files to your GitHub repository:

index.html news.html login.html contact.html dashboard.html bfx-admin.html ← NEW vercel.json

Vercel will auto-deploy in about 30 seconds. Your admin panel will be live at:

https://your-site.vercel.app/bfx-admin.html
Bookmark that URL. Share it with nobody. It's your private control room.
✓ You're All Set

Once configured, the full system works like this:

  • Student registers on login.html → saved to Firebase automatically
  • You open bfx-admin.html → enter password → see all students
  • Student pays & messages on WhatsApp → you click ⚙ Manage → upgrade tier → save
  • Student logs into dashboard.html → Firebase syncs their new tier → Pro/Apex content unlocks
  • Student lesson progress syncs back to Firebase → you can see it in the admin panel
